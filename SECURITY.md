# Security Policy

## Supported Versions

ORION is maintained on the current `main` branch and the latest published `0.1.x` release line.

| Version | Supported |
| ------- | --------- |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:       |

## Reporting a Vulnerability

If you believe you found a security issue in ORION, please report it privately and do not open a public issue.

Please include:
- A short description of the issue
- Steps to reproduce, if available
- The affected page, route, service, or dependency
- Any proof-of-concept details you are comfortable sharing

### Where to Report

Send reports to the project maintainer through one of the following:
- GitHub Security Advisories for the repository
- Direct maintainer contact if available in the repository profile or README

### What to Expect

- Acknowledgment within 3 business days
- An initial assessment within 7 business days
- A status update if the issue is accepted, declined, or needs more information

### Response Process

If the report is accepted, we will:
- Triage the severity
- Confirm affected versions
- Prepare and test a fix
- Release the fix or merge it to `main`
- Credit the reporter if they want public attribution

If the report is declined, we will explain why and, when possible, suggest a safer alternative or mitigation.

### Scope

Security reports are especially useful for:
- Authentication or authorization flaws
- Sensitive data exposure
- Injection issues
- Dependency vulnerabilities
- Unsafe file handling
- Broken access control
- Secrets accidentally committed to the repo

Thank you for helping keep ORION secure.
